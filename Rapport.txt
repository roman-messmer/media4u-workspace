🛠 ABSCHLUSS-RAPPORT: SERVER-INFRASTRUKTUR & SICHERHEIT

Ein sauberer Rapport ist für die Wartung eines komplexen Tech-Stacks mit mehreren Subdomains und Container-Strukturen unerlässlich. Hier ist die lückenlose Dokumentation der durchgeführten Arbeiten an der Server-Infrastruktur.

1. NGINX-ARCHITEKTUR & ROUTING
--------------------------------------------------------------------------------
Das System wurde von einem Docker-basierten Proxy-Setup auf eine native Host-NGINX-Architektur umgestellt, um maximale Performance und Kontrolle über statische Dateien zu erhalten.

• Globaler HTTP-Redirect: Jede Anfrage auf Port 80 wird nun permanent (301) auf HTTPS (Port 443) umgeleitet.
• Port-Konflikt gelöst: Die Docker-Container nginx-proxy und acme-companion, welche die Ports 80 und 443 blockierten, wurden gestoppt.
• Subdomain-Isolation: In der Hauptdomain media4u.ch wurde eine strikte Blockierung der physischen Unterordner (/projekte, /editor, /werbung, /diverses) via location ^~ eingerichtet, um den Zugriff ausschließlich über die dafür vorgesehenen Subdomains zu erzwingen.
• SPA-Routing & Fallback: Für alle React-basierten Projekte (projekte, werbung, editor) wurde ein robuster try_files-Fallback auf die index.html konfiguriert.
• Vite-Optimierung: Statische Assets erhalten nun optimierte Cache-Header (immutable, max-age=31536000), während die index.html zur Vermeidung von Deployment-Verzögerungen niemals gecacht wird.

2. SPEZIFISCHE PROJEKT-KONFIGURATIONEN
--------------------------------------------------------------------------------
• media4u.ch: React SPA; Haupt-Frontend, blockiert Unterordner-Zugriffe.
• editor.media4u.ch: React SPA; Passwortschutz via .htpasswd; 301-Redirect von / auf /vorlagen.
• diverses.media4u.ch: Pure Static; Kein SPA-Fallback; direkter Zugriff auf Ordner wie /game/ oder /360_umsicht/.
• cms.media4u.ch: API Proxy; Sicherer Reverse Proxy für den internen Strapi-Docker-Container.

3. SICHERHEIT & SSL-HÄRTUNG
--------------------------------------------------------------------------------
Die SSL-Konfiguration wurde nach modernen Senior-Standards gehärtet, um ein A+ Rating zu erreichen.

• Protokolle: TLS 1.0 und TLS 1.1 wurden global deaktiviert.
• Unterstützung: Es werden nur noch TLS 1.2 und 1.3 unterstützt.
• HSTS (Strict Transport Security): Der Browser wird für ein Jahr angewiesen, die Seite und alle Subdomains ausschließlich über HTTPS aufzurufen.
• Syntax-Update: Die veraltete listen 443 ssl http2 Direktive wurde auf das neue http2 on; Format aktualisiert, um NGINX-Warnungen zu eliminieren.
• Security Header: X-Frame-Options (Clickjacking-Schutz) und X-Content-Type-Options wurden global implementiert.

4. DEPLOYMENT-VORBEREITUNG
--------------------------------------------------------------------------------
• Dateipfade: Alle Projekte wurden auf das einheitliche Verzeichnis /srv/media4u/site/ umgestellt, was perfekt mit den PowerShell-Deploy-Skripten harmoniert.
• Berechtigungen: Das Verzeichnis für diverses.media4u.ch wurde manuell erstellt und mit den korrekten Rechten für den User roman und die Gruppe nginx versehen.
• Zertifikate: Ein neues Let's Encrypt Zertifikat für diverses.media4u.ch wurde via Certbot erfolgreich ausgestellt und in die automatische Erneuerung aufgenommen.

WICHTIGER HINWEIS FÜR DIE ZUKUNFT:
Da die Subdomain diverses kein SPA-Fallback nutzt, müssen alle Verlinkungen in der dortigen index.html exakt den Ordnernamen auf dem Server entsprechen (z. B. /360_umsicht/ statt /360°_umsicht/).
